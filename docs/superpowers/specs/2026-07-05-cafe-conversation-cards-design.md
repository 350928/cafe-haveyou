# 디자인 명세서: 카페 대화 카드 게임 (Have You?)

이 문서는 카페 테이블용 태블릿 및 모바일 브라우저에서 실행 가능한 터치 기반의 대화 카드 게임 웹 애플리케이션의 디자인 및 기능 명세를 정의합니다.

## 1. 개요 및 설계 목표
- **목적**: 카페에 방문한 친구, 연인, 혹은 1인 고객이 무작위 카드를 뽑아 서로를 더 잘 알아가고 재미있게 친분을 쌓을 수 있는 분위기 메이커 게임.
- **주요 타겟 환경**: 태블릿 및 모바일 기기의 브라우저 화면 (터치 최적화).
- **디자인 방향**: 부드럽고 친근하며 따뜻한 파스텔톤 컬러(Playful & Pastel)와 둥글둥글한 UI 요소, 그리고 직관적인 3D 카드 뒤집기 애니메이션 제공.
- **구현 아키텍처**: 서버나 데이터베이스가 필요 없는 **100% 클라이언트 사이드 정적 웹 앱** (HTML, CSS, JavaScript).

---

## 2. 사용자 인터페이스 (UI) 구성
애플리케이션은 화면 전환이 매끄러운 단일 페이지(Single Page App)로 구성되며, 크게 세 가지 뷰(View) 상태를 가집니다.

### A. 웰컴 화면 (Welcome Screen)
- **배경**: 부드러운 파스텔 옐로우와 살구색 그라데이션.
- **로고**: 큰 폰트와 둥글둥글한 라운딩 처리를 한 한글/영어 복합 타이틀 ("Have You? - 우리들의 대화 카드").
- **장식 요소**: 화면 중앙에 둥실둥실 떠 있는 듯한(Bouncing) 아기자기한 카드 덱 그래픽 일러스트.
- **버튼**: 화면 중앙 하단에 크게 자리 잡은 `[시작하기]` 버튼 (터치 시 스케일 감소 마이크로 피드백).

### B. 게임 플레이 화면 (Game Screen)
- **상단 바**:
  * ① 홈(처음으로) 아이콘 버튼.
  * ② 중앙: 현재 드로우 번호 표시 (예: "12번째 대화").
  * ③ 오른쪽: 게임 재설정(새로 시작) 아이콘 버튼.
- **카드 덱 영역 (Card Deck Layer)**:
  * 화면 중간에 카드의 뒷면(파스텔 패턴 디자인)이 겹겹이 쌓여 있는 카드 덱 시각화.
  * 카드 덱을 누르면 카드가 위로 회전하며 날아와 중앙에 펼쳐지는 애니메이션 실행.
- **활성화된 카드 (Active Card)**:
  * 크기: 모바일/태블릿 화면 너비의 약 85%를 차지하는 카드 형태.
  * 라운딩: 모서리에 24px 이상의 부드러운 Border Radius 적용.
  * 카테고리 태그 및 색상 테마:
    - **MBTI**: 파스텔 핑크 (#FFD1DC) / 💬 MBTI 태그
    - **취미/관심사**: 파스텔 민트 (#D1F2D9) / 🎨 취미 태그
    - **사는 곳**: 파스텔 옐로우 (#FFF5BA) / 📍 사는 곳 태그
    - **좋아하는/싫어하는 것**: 파스텔 퍼플 (#E8D3FC) / 💖 취향 태그
    - **직업/전공**: 파스텔 블루 (#D3EEFF) / 💼 일/공부 태그
  * 내용: 중앙에 가독성이 높은 굵은 폰트(20~24px)의 대화 질문 텍스트 배치.
- **하단 제어**: 
  * 활성화 카드 아래에 다음 카드를 바로 드로우할 수 있는 `[다음 카드 뽑기 ➔]` 버튼 배치.

### C. 스페셜 미션 오버레이 (Special Mission UI)
- **등장 조건**: 전체 카드 중 무작위 확률로 미션 카드가 뽑혔을 때 활성화.
- **비주얼**: 일반 카드와 완전히 다른 다채로운 파스텔 그라데이션 카드 배경, 반짝이는 별 그래픽 효과 및 상단의 **"🔥 SPECIAL MISSION"** 하이라이트 배지.
- **내용**: 가벼운 벌칙이나 아이스브레이킹 미션 텍스트 제공.

## 4. 제어 로직 및 상태 관리
- **카드 셔플 (Shuffle)**: 게임 로드 혹은 재시작 시, 카드 데이터베이스 복사본을 피셔-예이츠(Fisher-Yates) 알고리즘으로 무작위 셔플합니다.
- **카드 진행 상태 추적**:
  - `remainingDeck`: 아직 뽑지 않은 카드 배열.
  - `drawnCards`: 이미 뽑아서 보여준 카드 배열.
- **자동 덱 리셋**: `remainingDeck`이 비게 되면 "모든 카드를 뽑았습니다! 다시 섞습니다." 라는 오버레이 팝업 메시지를 노출하고, 1.5초 후 자동으로 카드를 리셋하여 처음부터 다시 무작위로 뽑을 수 있도록 처리합니다.
- **브라우저 상태 유지 (Local Storage)**:
  - 브라우저를 새로고침하거나 기기를 잠시 잠금 상태로 전환했다가 켜도 기존 카드 드로우 순서와 뽑은 카드 정보를 그대로 유지하도록 `localStorage`에 상태를 동기화합니다.

## 5. 인터랙션 및 애니메이션 가이드
- **3D Card Flip**: 카드를 드로우할 때 CSS `perspective` 속성을 활용해 0.5초 동안 뒤집어지며 커지는 3D 플립 애니메이션 구현.
- **Bouncing Animation**: 대기 화면의 카드 더미에 `@keyframes bouncing` 효과를 부여해 위아래로 둥실둥실 떠오르는 부드러운 움직임 연출.
- **Fade In / Out**: 화면 상태 전환(스타트 화면 <-> 메인 플레이 화면) 시 전체 화면 영역에 CSS `opacity` 전환 효과를 가미하여 부드러운 흐름 형성.
